Let's learn Git & GitHub today.

What is Git?

Git is a distributed version control system. It tracks changes in files and helps manage code history.

What is GitHub?

GitHub is a cloud platform that hosts Git repositories. It helps teams collaborate, review code, and manage projects.

Basic Git Workflow

Working Directory → Staging Area → Commit History

Important Git Commands

git init

Initializes a new Git repository in the current folder.

git status

Shows the current state of files. Displays staged, unstaged, and untracked files.

git add

Moves selected changes to the staging area. Prepares files for commit.

git commit

Creates a snapshot of staged changes. Saves them permanently in the repository history.

git log

Displays the commit history. Shows commit IDs, authors, and messages.

git branch

Lists all branches. Helps manage parallel development.

git checkout

Switches between branches or commits.

git merge

Combines changes from one branch into another.

git clone

Creates a local copy of a remote repository.

git pull

Fetches and merges changes from the remote repository.

git push

Uploads local commits to the remote repository.

Key Points to Remember

  • Git tracks changes, not files alone
  • Commits are permanent snapshots
  • Staging controls what goes into a commit
  • GitHub is for collaboration
