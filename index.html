Let's learn Git & GitHub today.

What is Git?

Git is a distributed version control system (VCS). It allows multiple people to work on the same codebase by tracking and managing changes to files.

What is a Version Control System (VCS)?

A version control system records every change made to a file. It allows you to view history, restore previous versions, and identify who made specific changes.

Key Features of Git

  • Backup and Restore, files are protected against accidental loss
  • Collaboration, multiple users work simultaneously on the same project
  • Branching and Merging, users experiment safely and merge changes later
  • Change Tracking, view what changed and who changed it

Git Configuration

Configure username and email

Identifies the user making commits.

git config --global user.name "username"

git config --global user.email "mailid"

View configuration

git config --list

Initializing and Tracking

git init

Initializes a new Git repository.

git status

Shows the current state of the working directory and staging area.

git add

Adds files to the staging area.

git add filename

git add .

git commit

Creates a permanent snapshot of staged changes.

git commit -m "commit message"

Git Workflow

Working Directory → Staging Area → Local Repository

History and Comparison

git log

Displays commit history.

git diff

Shows differences between older and newer versions.

git show

Displays detailed information about a commit.

git show commit_id filename

git checkout (file)

Restores a specific file from a commit.

git checkout commit_id -- filename

git checkout (branch)

Switches branches or restores the latest version.

git checkout branch_name

Undo and Recovery Cases

Discard working directory changes

git restore filename

Unstage files

git restore --staged file_path

View staged changes

git diff --cached

Restore both staging and working tree

git restore --worktree index.html

Undo commits

git reset --soft HEAD^

git reset --hard HEAD^

Advanced git log Options

git log -p -n

git log --stat

git log --pretty=oneline

git log --pretty=format:"%h %an %ar %s"

git log -S function_name

git log --grep="search"

git log --since="2024-01-01"

git log --until="2024-12-31"

git log --author="Keshav"

git log --no-merges

Remote Repository

A remote repository is a version of your project hosted on the internet or a network server.

git clone

Creates a local copy of a remote repository.

git clone https_link_from_github

git push

Pushes local commits to the remote repository.

git pull

Fetches and merges changes from remote to local.

Branching and Merging

git branch

Lists branches or shows current branch.

Create and switch branch

git checkout -b branch_name

Switch branches

git checkout branch_name

Merge branches

git merge new_branch

Forking

Forking creates a personal copy of another user’s repository to modify and propose changes via pull requests.

.gitignore

Specifies files and folders Git should ignore. Useful for sensitive or unnecessary files.

Cleaning Repository

git clean -n

git clean -f

Git Tags

Tags mark specific commits with meaningful identifiers.

git tag -a tag_name -m "message"

git show tag_name

git tag -a tag_name commit_id

git tag -d tag_name

git push origin tag_name

git push origin --tags

Key Takeaways

  • Git tracks changes, not just files
  • Commits are permanent checkpoints
  • Staging controls what goes into history
  • Branches enable safe experimentation
  • GitHub enables collaboration at scale
